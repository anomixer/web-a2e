Help & Documentation

Getting Started

Welcome to the Apple //e Emulator! This web-based emulator faithfully recreates the Apple //e Enhanced computer from 1983, allowing you to run classic Apple II software directly in your browser.

Quick Start

  1. Click the Power button to turn on the emulator
  2. Click on the screen to give it keyboard focus
  3. Insert a disk image using the Insert button on either drive
  4. Type PR#6 and press Return to boot from drive 1

What is the Apple //e?

The Apple //e (Enhanced) was Apple's most popular Apple II model, featuring 128KB of RAM, 80-column text display, and double hi-res graphics. It was the workhorse of the Apple II line and ran thousands of educational, productivity, and entertainment programs.

Tip: Press F1 at any time to open this help documentation.

Text Selection & Copy

You can select and copy text directly from the emulator screen:

  • Click and drag on the screen to select text
  • Selected text is automatically copied when you release the mouse
  • Press Ctrl+C (or Cmd+C on Mac) while selecting to copy

Emulator Features

This emulator provides an accurate recreation of the Apple //e Enhanced with modern enhancements.

65C02 CPU

Accurate emulation of the 65C02 processor at 1.023 MHz with all instructions and timing.

WebGL Rendering

GPU-accelerated display with authentic CRT effects, scanlines, and color fringing.

Audio Emulation

Authentic speaker clicks with optional drive motor and head seek sounds.

Clipboard Support

Paste text directly into the emulator and copy text from the screen.

Display Modes

  • Text Mode - 40 or 80 column text display
  • Lo-Res Graphics - 40x48 or 80x48 color blocks
  • Hi-Res Graphics - 280x192 with 6 colors
  • Double Hi-Res - 560x192 with 16 colors
  • Mixed Mode - Graphics with 4 lines of text

Display Settings

Customize the visual appearance of the emulator display through the Display Settings panel.

CRT Effects

  • Scanlines - Authentic horizontal scan line effect
  • Screen Curvature - Simulates curved CRT glass
  • Vignette - Darkening at screen edges
  • Bloom/Glow - Phosphor glow simulation

Analog Effects

  • Color Fringing - NTSC color artifact emulation for Hi-Res graphics
  • Noise - Subtle signal noise

Image Adjustments

  • Brightness - Overall image brightness
  • Contrast - Color contrast level
  • Saturation - Color intensity

Sharp Pixels Mode

Disable all effects for a clean, pixel-perfect display. Useful for screenshots or if you prefer a modern look.

Note: You can resize the display by dragging the corner handles on the monitor bezel.

Inserting Disks

  • Insert button - Opens a file picker to select a disk image
  • Recent button - Shows the last 20 disk images used, click to quickly re-insert
  • Drag and drop - Drag disk files directly onto a drive
  • Blank button - Creates a new blank formatted disk

Ejecting Disks

Click the Eject button to remove a disk. If the disk has been modified, you'll be prompted to save it.

Booting from Disk

After inserting a bootable disk in drive 1:

  • Type PR#6 and press Return to boot
  • Or press the Reboot button to cold boot

Disk Persistence

The emulator remembers your disk setup between sessions:

  • Disks in each drive are automatically restored when you return
  • Recent disks are stored locally so they work even if the original file is moved

Important: Modified disks are stored in memory only. Use Eject and save when prompted to preserve changes.

State Management

The emulator can save and restore its complete state, allowing you to pause your session and resume exactly where you left off.

What Gets Saved

  • CPU State - All registers (A, X, Y, SP, PC) and status flags
  • Memory - Full 128KB RAM (64KB main + 64KB auxiliary)
  • Language Card - 16KB Language Card RAM banks
  • Soft Switches - All memory banking and display mode settings
  • Disk Drives - Complete disk images including any modifications, track positions, and filenames

Auto-Save

When enabled (the default), the emulator automatically saves state every 5 seconds while running. This means you can close your browser and return later to continue exactly where you left off.

  • Toggle auto-save on/off in the State popup
  • State is stored in your browser's local storage
  • A timestamp shows when the last save occurred

Manual Save & Restore

  • Save Now - Immediately saves the current state
  • Restore - Reloads the last saved state (performs a full power cycle)

How Restore Works

Restoring state is equivalent to powering off the emulator, powering it back on, and then loading the saved state. This ensures a clean restoration with no leftover state from the current session.

Tip: State is preserved even if your browser closes unexpectedly. Just return to the emulator and click Restore to continue.

Note: State is stored locally in your browser using IndexedDB. Clearing browser data will remove saved states.

Debug Tools

Advanced debugging tools for developers and enthusiasts. Access these from the Debug menu.

View CPU registers, flags, and current instruction. Step through code one instruction at a time.

Drive Monitor

Watch disk drive activity including track position, motor status, and read/write operations.

Soft Switches

Monitor the state of Apple II soft switches that control memory banking, display modes, and I/O.

Memory Browser

Examine and search through the full 128KB memory space with hex and ASCII views.

Memory Heat Map

Visualize memory access patterns in real-time. See which areas are being read or written.

Stack Viewer

Monitor the 6502 stack at page $01, showing push/pop operations and return addresses.

Zero Page Watch

Track changes to zero page memory ($00-$FF), commonly used for variables and pointers.

Tips & Troubleshooting

Common Issues

Keyboard not working?
Click on the emulator screen to give it focus. The green border indicates the screen has keyboard focus.

No sound?
Browser security requires user interaction before playing audio. Click the Power button or anywhere on the page first.

Disk won't boot?
Make sure the disk is in Drive 1 and type PR#6 then press Return. Some disks may need a cold boot (Reboot button).

Screen looks wrong?
Try the "Sharp Pixels" option in Display Settings, or adjust the CRT effect sliders to your preference.

Performance Tips

  • Close other browser tabs to ensure smooth emulation
  • Use Chrome or Firefox for best WebGL performance
  • Disable browser extensions that might interfere with keyboard input
  • If audio stutters, try reducing other system load
